# Core Concepts

Version: 2.0.0-alpha

Status: Frozen

---

# Employee

The product abstraction visible to users.

An Employee performs work, learns continuously and collaborates with other Employees.

Internally, every Employee is implemented as one Workspace.

---

# Workspace

The fundamental runtime unit of AIWorkbench.

A Workspace owns:

- Memory
- Knowledge
- Context
- Tasks
- Events
- Artifacts
- Capabilities

Workspace is the permanent digital asset.

Employees are the product representation of Workspaces.

---

# Brain

The reasoning engine.

A Brain never owns data.

It only reads from and writes to a Workspace.

Brains are replaceable.

Examples:

- Ollama
- OpenAI
- Claude
- Gemini
- DeepSeek

---

# Capability

A reusable skill.

Examples:

- Read PDF
- Search Browser
- Git Commit
- Generate Report
- Analyze Screenshot

Capabilities are composable.

---

# Task

A goal assigned to an Employee.

A Task may produce:

- Events
- Artifacts
- Reports
- Decisions

---

# Event

A record describing something that happened.

Examples:

- User instruction
- File uploaded
- Screenshot received
- Browser completed
- Tool failed

Events drive the cognitive lifecycle.

---

# Artifact

A work result produced by an Employee.

Examples:

- Markdown
- Report
- Code
- Image
- Spreadsheet
- Deployment

Artifacts are permanent outputs.

---

# Memory

Long-term information accumulated by a Workspace.

Memory is never generated by the Brain alone.

It is accumulated through work.

---

# Knowledge

Structured experience extracted from Memory.

Knowledge is reusable.

Knowledge improves future decisions.

---

# Provider

The underlying LLM implementation.

Providers are infrastructure.

They are never part of the business model.
